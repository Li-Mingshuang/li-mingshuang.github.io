Using Git for Version Control
（*[Eric_Matthes]_Python_Crash_Course 2nd* - Appendix D ）

Configuring Git

  • 初次使用时需要配置用户信息
  • 邮箱可以随便写

Ignoring File

  • .gitignore 文件
    • 文本文件
    • Git 会忽略该文件中记录的文件和文件夹
    • 将 __pycache__/ 添加到该文件中，使得 Git 忽略该文件夹中所有文件
    • 编辑器可能会忽视 . 开头的文件，如果是这样需要设置
  • __pycache__ 文件夹
    • 用于存放编译器编译 .py 文件自动生成的 .pyc 文件，这些文件不需要用 Git 追踪

Initializing a Repository

  • 打开终端，导航到项目文件夹，运行以下指令初始化 Git 仓库（repository）
  • 不知道这个终端指代 windows 上的什么工具，我都是在文件夹中右键，然后 git bash here，来打开命令窗口
  • 该命令会在目录中创建一个文件夹 .git\ ，该文件夹包含 git 管理仓库的所有文件
  • git 仓库：指代项目中 git 追踪的文件集合

Checking the Status

git_practice$ git status

Adding Files to the Repository

git_practice$ git add .
git_practice$ git add [file]

  • 添加后，git 开始追踪这些文件

Remove files from the Repository

  • git rm --cached [file]
    • 用于从暂存区（index）中删除指定的文件，但保留工作目录中的文件
  • 对于移除后不需要跟踪的文件，应该手动添加到.gitignore文件中

Making a Commit

获得一个项目当前的快照，后续修改出错后可以恢复到这个状态。

首次提交

  • 添加（追踪）所有文件 ：git add .
  • 提交添加的所有文件： git commit -m "Started project."
    • The -m flag tells Git to record a message in the log for this commit.
  • 提交后检查状态，期望看到这样的信息：
  • 否则需要检查是否存在问题，比如提交前没有添加文件。

第二次提交

  • 对于修改后的文件有两种提交方式
    1. 使用 git add [file] 或 git add . 添加该文件，然后提交
    2. 直接通过 git commit -am "change and commit" 提交
      • The -a flag tells Git to add all modified files in the repository to the current commit.
      • 这个命令无法提交新的文件
  • 对于新的文件，需要使用 git add 添加该文件然后提交

Checking the Log

  • Git 记录了所有提交的 log
  • 检查 log：
    • 显示当前分支所有提交 过的快照信息
    • 完整 log 查询：git log
      • ID：每次提交时 Git 产生的一个独特的 40 位 ID
      • 提交者
      • 提交时间
      • 提交备注
    • 简洁 log 查询：git log --pretty=oneline
      • 只包含 ID 和提交备注
  • 对于比较长的 log，Git 命令窗口只显示前面的部分信息，按 enter 出现新的一行内容，按 q 退出

Reverting a Change

  • git restore .
    • 将所有跟踪文件中的修改恢复至上次提交的状态
    • 不会删除新的文件（还未跟踪）
  • restore 已经取代教程中使用的 checkout 命令，但是后者目前还是能得到同样的效果

Checking Out Previous Commits

  • checkout

    • 用于访问
      • 当前分支所有提交过的状态：git checkout [6位ID]
      • 其他分支：git checkout [其他分支名称]
    • 访问旧的状态时属于脱离 HEAD 的状态（detached HEAD state），因为 Head 指向当前分支的最新状态节点
      • 此时可以修改、提交代码并创建新的分支
        • 仅修改并提交不会影响存在的任何分支，如果回到其他分支，这些修改不会被保留 (似乎还是有这些节点信息，因为此时 git 会询问是否为这个状态节点创建新分支)
        • 想要保留需要创建新的分支 git switch -c <new-branch-name>
          • 撤销创建 git switch -
    • 书中建议如果不使用这些更高级的功能，访问旧的状态时不要修改代码，即新手只要会保存节点和回退即可
    • 回到 main 分支的 HEAD 节点： git checkout main

  • git reset --hard [6位ID]

    • 将代码重置为某个提交（丢弃最近的提交，将代码恢复到之前的某个状态）
    • 恢复后的状态是当前分支的 HEAD 节点

We first check the status to make sure we’re on the master branch
We then issue the git reset --hard command with the first six characters of the reference ID of the commit we want to revert to permanently

Unless you want to work with some more advanced features of Git, it’s best not to make any changes to your project when you’ve checked out an old commit.

Deleting the Repository

  • 删除版本信息
    • 不会影响当前的代码
    • 方法
      1. 直接删除 .git 文件夹
      2. 使用 linux 命令行（git bash）rm -rf .git
      3. 使用 windows 命令行 rmdir /s .git
  • We first check the status and see that we have a clean working tree.
    • ? working tree 是不是 clean 的有什么关系，反正都要删除 git 库了
