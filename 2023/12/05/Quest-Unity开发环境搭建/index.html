Quest-Unity开发环境搭建

Li Mingshuang
2023-12-05 (Updated: 2023-12-05)
VR

需安装软件

  • 主要
    • oculus
    • unity
      • lts（长期支持版本）
      • android支持套件
  • 辅助
    • sidequest
    • oculus develper hub
  • 软件设置
  • oculus
    • 用于直接串流unity开发场景到头显中，而不用打包发布
    • 设置 > 测试版 > 打开 开发者runtime，oculus link 透视
    • 设置 > 通用> 打开未知来源，openXR runtime设置成“将Oculus设为使用中”

创建Unity项目

  1. 选择需要的版本
  2. 选择3D模块
  3. 修改项目名称和存放路径

Unity界面介绍

  • 菜单栏（顶部）
  • Scene窗口
    • 看到游戏窗口
    • 在该窗口中编辑游戏物体
  • Hirrarchy窗口
    • 显示当前场景中的游戏物体以及物体层级
    • 层级关系：
      • 首先有个场景
      • 场景中有些物体
      • 每个物体上有些组件，这些组件赋予物体在游戏中的行为
        • 摄像头
        • 碰撞体
  • Inspector窗口
    • 显示物体挂载的组件或者脚本
  • Project窗口
    • 显示项目文件夹
  • Console窗口
    • 游戏运行时输出日志信息
    • 用来调试的工具
  • Game窗口
    • 点击运行后，game窗口显示游戏运行时的画面

Quest一体机开发的环境配置

  • 菜单栏 > file > build settings，切换平台到android
  • 导入xr开发的sdk
    • 先介绍手势追踪和mr开发，所以选择sdk是meta官方的Meta XR UPM Packages
    • 菜单栏 > window > assets store > search online > 搜索该插件 > 添加至我的资源
    • 菜单栏 > window > package manager > packages切换成my assets > 找到sdk，点击import > show assets，clean up
    • 导入成功后可以在assets文件夹中看到oculus文件夹
  • 设置
    • 菜单栏 > edit > project settings >
      • Oculus > 点击fix all和apply all自动配置开发quest一体机需要的设置，直到全部变绿
      • XR Plug-in management > 确保在安卓模块下勾选了Oculus，在Pc端勾选Oculus或Open XR
        • Oculus > PC端把single pass改成multiple pass，安卓端选择对应的target devices（quest+quest2）
      • Player > 安卓端 > other settings >
        • rendering > colorspace: Linerar
        • Identification > Mininum API Level: Android 10.0
        • Configuration >
          • Scripting Backend: IL2CPP
          • Target Arch > 勾选ARM64，取消勾选ARM v7
      • Quality > Anti Aliasing: 4x
    • 启用手势追踪
      • Projects窗口 > Assets > Oculus文件夹 > Oculus Project Config > Hand Tracking Support: 设为包含手势的选项，手势追踪频率：high，手势追踪版本：v2
  • 展示示例
    • 打开运行可以直接在头显中显示
    • 发布到一体机
      • 菜单栏 > File > Build Settings >
        • 需要将场景添加到Scenes In Build，点击Add Open Scenes可以直接把当前场景添加进去
        • Run Device改成连接的头显
        • 点击Build and run
        • 会打包成一个apk，然后发布到头显中直接运行
        • 需要创建一个文件夹，可以在项目路径创建一个Build文件夹存放
